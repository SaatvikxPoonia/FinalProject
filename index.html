Project Introduction and Overview

This B.Tech project, conducted during a 6-month internship at Eternal (formerly Zomato) from January to June 2025, focuses on enhancing content discovery mechanisms through sophisticated A/B testing methodologies and deep link resolution systems.

Project Context

Working within the Search and Discovery team of the District vertical, this project addressed critical challenges in content personalization and marketing attribution. The existing system suffered from inefficient database operations and lacked sophisticated ranking mechanisms for content discovery.

Key Achievements Summary

  • Cost Optimization: Achieved a remarkable 97.3% reduction in DynamoDB read costs, decreasing from over 128 million RCUs to under 3.5 million RCUs daily, resulting in tens of thousands of rupees in monthly savings.
  • Machine Learning Integration: Successfully replaced static dashboard configurations with ML-based rail ranking systems that adapt to user behavior and content performance metrics.
  • Attribution System: Built a comprehensive deferred deep link resolution system that significantly improved app attribution for marketing campaigns.
  • Experimental Framework: Conducted 4 major A/B experiments with measurable engagement improvements across different content categories.
  • Scalable Architecture: Designed and implemented microservices architecture using gRPC and worker pools for high-throughput operations.

Technical Environment

The project leveraged a modern tech stack including Go (Golang) for backend services, Amazon DynamoDB and Redis for data storage and caching, Google Ads API and AppsFlyer for marketing integrations, and comprehensive monitoring through Grafana with structured logging and distributed tracing.

Project Significance

This work demonstrates the practical application of machine learning and experimental design in production systems, showcasing how data-driven approaches can simultaneously improve user experience and reduce operational costs. The project bridges theoretical computer science concepts with real-world business impact, making it an ideal capstone for B.Tech studies.

Technical Architecture Deep Dive

The system architecture follows microservices principles with emphasis on scalability, maintainability, and performance optimization. The design accommodates high-throughput requirements while maintaining system reliability and cost efficiency.

Modular Discovery Framework

The core discovery framework implements a ranked entity serving system that processes millions of content requests daily. The framework utilizes a plugin-based architecture allowing for easy integration of new ranking algorithms and content types without affecting the core system stability.

ML-based Ranking Engine

The machine learning component replaced traditional static configurations with dynamic ranking based on real-time feature extraction. Features include user engagement metrics, content freshness, geographic relevance, and temporal patterns. The ranking engine processes features in real-time using efficient in-memory computations and cached model predictions.

A/B Testing Infrastructure

A robust experimentation platform was developed to support week-on-week metrics analysis. The system includes experiment configuration management, traffic allocation algorithms, statistical significance testing, and automated reporting. The platform ensures proper randomization and maintains experiment isolation to prevent cross-contamination.

Google Ads Integration Pipeline

The integration pipeline handles OAuth authentication, rate limiting, and bulk data processing for Google Ads campaigns. Worker pools manage concurrent API requests while respecting API quotas. The system includes retry mechanisms, error handling, and comprehensive logging for debugging and monitoring.

Multi-layer Caching Strategy

A sophisticated caching architecture combines in-memory caching for frequently accessed data with Redis for distributed caching across service instances. Cache invalidation strategies ensure data consistency while maximizing hit rates. The caching layer contributed significantly to the 97.3% reduction in database read operations.

Performance Optimization Techniques

Several optimization techniques were implemented including connection pooling, batch processing, asynchronous operations, and efficient data serialization. Database queries were optimized using proper indexing strategies and query pattern analysis. The system architecture supports horizontal scaling through stateless service design.

A/B Testing Methodology and Implementation

The A/B testing framework represents a comprehensive approach to experimental design in production systems, enabling data-driven decision making for content discovery optimization.

Experimental Design Principles

All experiments followed rigorous statistical design principles including proper sample size calculation, randomization techniques, and multiple hypothesis testing corrections. Each experiment was designed with clear success metrics, minimum detectable effects, and statistical power analysis to ensure meaningful results.

CTR-based Optimization (exp1-cg)

The control group experiment focused on click-through rate optimization for rail positioning. This baseline approach used historical CTR data to rank content rails, providing a benchmark for comparison with more sophisticated ranking methods. The implementation tracked click events, calculated CTR metrics, and adjusted rankings based on sliding window calculations.

OTR-based Ranking (exp1-tg)

The target group implemented order-through rate ranking to optimize for actual conversions rather than just clicks. This approach considered the complete user journey from content discovery to purchase completion. The system tracked conversion funnels, calculated OTR metrics, and weighted rankings based on revenue impact rather than engagement alone.

Geographic Personalization (exp2-tg)

This experiment utilized S2 cell-based geographic indexing at depth 13 for location-aware content personalization. The S2 geometry library provided efficient spatial indexing, enabling real-time geographic matching. User locations were mapped to S2 cells, and content was ranked based on geographic relevance, local popularity, and regional preferences.

City-wise Property Ranking

The city-based ranking experiment leveraged property metadata signals to customize content ordering. This approach considered city-specific characteristics such as demographics, economic indicators, cultural preferences, and seasonal patterns. The ranking algorithm incorporated these signals through feature engineering and weighted scoring mechanisms.

Statistical Analysis and Results Interpretation

Results were analyzed using appropriate statistical tests including chi-square tests for categorical outcomes and t-tests for continuous metrics. Confidence intervals were calculated for all measurements, and multiple comparison corrections were applied where necessary. The analysis included both statistical significance and practical significance evaluation.

Lessons Learned from Experimentation

The experimental process revealed important insights about user behavior, content preferences, and system performance trade-offs. Some experiments showed unexpected results, highlighting the importance of rigorous testing before implementing changes in production systems.

Deep Link Resolution System

The deep link resolution system addresses critical attribution challenges in mobile marketing campaigns by creating a seamless connection between advertising campaigns and app content.

Problem Statement and Motivation

Traditional app attribution systems often fail to provide accurate tracking for advertising campaigns, leading to inefficient marketing spend and poor user experience. With over 2 crore unique ads in the system and initially only ~28,000 having valid deep links, there was a significant opportunity for improvement in attribution coverage.

Deferred Deep Link Architecture

The deferred deep link system implements a sophisticated matching algorithm that connects advertising campaigns with app content even when direct deep links are unavailable. The system uses content metadata, campaign parameters, and machine learning techniques to establish these connections probabilistically.

Google Ads API Integration

The integration with Google Ads API required careful handling of OAuth authentication, API rate limits, and bulk data processing. The system implements robust error handling, retry mechanisms, and monitoring to ensure reliable data synchronization. Campaign data is processed in batches using worker pools to optimize throughput while respecting API constraints.

AppsFlyer Attribution Integration

AppsFlyer integration enables comprehensive attribution tracking across multiple marketing channels. The system forwards attribution events, handles callback processing, and maintains attribution data consistency. Real-time event processing ensures timely attribution reporting for marketing optimization.

Content Matching Algorithms

The content matching system uses multiple signals including text similarity, category matching, geographic relevance, and temporal patterns to establish connections between ads and app content. Machine learning models are trained on historical successful matches to improve matching accuracy over time.

Performance and Scale Considerations

The system processes millions of attribution events daily, requiring efficient data structures and algorithms. Caching strategies reduce latency for frequent lookups, while batch processing optimizes throughput for bulk operations. The architecture supports horizontal scaling to handle growing attribution volume.

Impact on Marketing Effectiveness

The improved attribution system enables more accurate measurement of marketing campaign performance, leading to better budget allocation and campaign optimization. Enhanced deep link coverage improves user experience by directing users to relevant content within the app.

Database Optimization and Cost Reduction

The database optimization project achieved a remarkable 97.3% reduction in DynamoDB read costs through systematic analysis and architectural improvements.

Initial Performance Analysis

The system initially consumed over 128 million Read Capacity Units (RCUs) daily, resulting in significant operational costs. Performance analysis revealed inefficient query patterns, lack of appropriate caching, and suboptimal data access patterns as primary cost drivers.

Query Pattern Optimization

Detailed analysis of database access patterns identified opportunities for query optimization. Hot partitions were redistributed, composite keys were restructured for better access patterns, and frequently accessed data was denormalized for single-query retrieval. These changes significantly reduced the number of database operations required for common use cases.

Multi-layer Caching Implementation

A sophisticated caching architecture was implemented with multiple layers:

  • Application-level caching: In-memory caches for frequently accessed data with configurable TTL and LRU eviction policies.
  • Distributed caching: Redis clusters for shared cache across service instances, enabling cache consistency and reducing cold start penalties.
  • Query result caching: Caching of expensive query results with intelligent invalidation based on data freshness requirements.

Worker Pool Architecture

The implementation of worker pool patterns optimized concurrent database operations. Connection pooling reduced connection overhead, while batch processing minimized the number of round trips to the database. Worker pools were sized based on database capacity and application requirements to maximize throughput without overwhelming the database.

Data Structure Optimization

Data structures were redesigned to minimize storage requirements and optimize access patterns. Sparse indexes were implemented for optional fields, and data compression techniques reduced storage costs. Schema design followed DynamoDB best practices for partition key distribution and sort key utilization.

Monitoring and Alerting

Comprehensive monitoring was implemented to track database performance metrics, cache hit rates, and cost trends. Grafana dashboards provide real-time visibility into system performance, while alerts notify operators of anomalous patterns that could indicate performance regressions.

Cost-Benefit Analysis

The optimization efforts resulted in monthly savings of tens of thousands of rupees while maintaining system performance and reliability. The return on investment was immediate, with the reduced operational costs significantly outweighing the development effort required for optimization.

Machine Learning Integration and Ranking Algorithm

The machine learning integration project replaced static dashboard configurations with dynamic, ML-driven rail ranking systems that adapt to user behavior and content performance in real-time.

Feature Engineering Pipeline

A comprehensive feature engineering pipeline extracts relevant signals from user interactions, content metadata, and contextual information. Features include user engagement metrics (clicks, time spent, conversion rates), content characteristics (freshness, popularity, category), geographic signals, and temporal patterns. Feature engineering includes normalization, encoding categorical variables, and creating interaction features.

Real-time Feature Extraction

The system implements real-time feature extraction to ensure ranking decisions are based on current data. Streaming data processing pipelines compute features with low latency, while feature stores maintain feature consistency across different services. The architecture supports both batch and real-time feature computation depending on feature characteristics and latency requirements.

Ranking Algorithm Development

The ranking algorithm uses machine learning models trained on historical engagement data to predict content relevance for individual users. Multiple model architectures were evaluated including linear models, gradient boosting, and neural networks. The final implementation uses ensemble methods combining multiple models to improve prediction accuracy and robustness.

Model Training and Validation

Model training follows rigorous machine learning practices including proper train/validation/test splits, cross-validation, and hyperparameter optimization. Feature importance analysis ensures model interpretability, while regularization techniques prevent overfitting. Model performance is evaluated using relevant metrics such as precision, recall, and ranking quality measures like NDCG.

A/B Testing for ML Models

The ML ranking system was validated through controlled A/B experiments comparing ML-based rankings with existing static configurations. Experiments measured user engagement metrics, conversion rates, and overall user satisfaction. Statistical analysis ensured that observed improvements were statistically significant and practically meaningful.

Production Deployment and Monitoring

The ML system was deployed using robust MLOps practices including model versioning, gradual rollout, and performance monitoring. Model performance is continuously monitored for drift detection, and retraining procedures ensure model freshness. The system includes fallback mechanisms to maintain service availability even if ML components fail.

Personalization and Geographic Adaptation

The ML system incorporates personalization signals to customize rankings for individual users and geographic regions. User behavior patterns, location-based preferences, and regional content popularity influence ranking decisions. The system balances personalization with content diversity to avoid filter bubbles while improving relevance.

Impact on Content Discovery

The ML-driven ranking system significantly improved content discovery effectiveness, leading to increased user engagement and satisfaction. The dynamic nature of ML rankings allows the system to adapt to changing user preferences and content trends, providing a more responsive and relevant user experience.

Performance Results and Business Impact Analysis

The project delivered measurable improvements across multiple business metrics while achieving significant cost reductions and operational efficiencies.

A/B Test Results by Content Category

Detailed analysis of A/B test results reveals varying performance across different content categories:

  • Artist Content: 0.38% improvement in engagement metrics, indicating modest but positive impact of the new ranking algorithm for artist-related content.
  • Event Content: 9.39% improvement, the highest performing category, suggesting that ML-based ranking is particularly effective for event discovery and recommendation.
  • Movie Content: 0.57% improvement, showing consistent positive impact similar to artist content.
  • Gig Content: -0.34% decrease, indicating that the optimization had trade-offs for this content type, possibly due to different user behavior patterns.
  • Restaurant Content: -10.00% decrease, the most significant negative impact, suggesting that restaurant discovery may require different optimization approaches.

Cost Reduction Analysis

The database optimization efforts yielded exceptional results with a 97.3% reduction in DynamoDB read costs. This improvement translates to tens of thousands of rupees in monthly savings, demonstrating the business value of technical optimization. The cost reduction was achieved without compromising system performance or reliability, making it a clear operational success.
